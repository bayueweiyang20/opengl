# 着色器类
编写、编译、管理着色器是件麻烦事。写一个类可以让代码结构清晰一点，它可以从硬盘读取着色器，然后编译并链接它们，并对它们进行错误检测，这就变得很好用了。这也会让你了解该如何封装目前所学的知识到一个抽象对象中。  
在我们的着色器类（Shader）中，有程序ID变量，构造函数参数为顶点着色器和片段着色器的路径，用于读取和构建着色器，类函数包括激活函数和uniform工具函数。  
这里需要使用.vs和.fs文件存储两个着色器的源码，需要扩展如下：  
Shader languages support for VS Code：提供GLSL语法高亮代码提示和错误检查  
GLSL Linter：帮助检测GLSL代码中的潜在问题  
GLSL Tools：支持着色器编译和运行的功能
